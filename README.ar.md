## ✨ ما الذي يستطيع Rethrone فعله

> 🪶 **تصميم بسيط** — يركّز على الطباعة والمساحات والخطوط الواضحة. لا شيء يشتّت الانتباه.

> 🧊 **Liquid Glass والسمات** — خمس سمات لونية أنيقة (أصفر، أزرق، أحمر، أخضر، بنفسجي)، و**عجلة ألوان** حرّة لأي لون تريده، وتأثير *Liquid Glass* العصري.

> 🌗 **تباين ديناميكي** — وضعا النص الداكن والفاتح لأفضل وضوح على أي خلفية.

> 🔤 **تحكّم كامل في الطباعة** — خطوط مخصّصة، وسماكة الخط، وأحجام أيقونات ونصوص قابلة للضبط عالميًا.

> 🖼️ **استوديو الخلفيات** — اضبط خلفية وقُصّها مباشرةً داخل المشغّل (uCrop).

> ✒️ **أيقونات خطية مخصّصة** — أيقونات Lucide بسيطة مدمجة لمظهر متناسق.

> 🎬 **حركة العودة** — حركات عالية الجودة شبيهة بنظام iOS تُعيد الأيقونات تمامًا إلى موضعها الأصلي.

<details>
<summary><b>🏠 عناصر واجهة قابلة للتهيئة على الشاشة الرئيسية</b> — اضغط للتوسيع</summary>

<br/>

- يمكن تشغيل/إيقاف **الساعة والتقويم (التاريخ) والطقس** كلٌّ على حدة.
- يقع **الطقس** افتراضيًا في **الزاوية العلوية اليمنى**، أسفل شريط الحالة مباشرةً.
- كل عنصر **قابل للتحريك بحرية** في وضع تحرير الشاشة الرئيسية (سحب وإفلات).
- تبقى الساعة **متزامنة** مع وقت النظام وتتحدّث تمامًا عند بداية كل دقيقة.

</details>

<details>
<summary><b>🎬 تحكّم دقيق في الحركات</b> — اضغط للتوسيع</summary>

<br/>

- **مفتاح رئيسي** لتشغيل أو إيقاف جميع الحركات.
- **قائمة فرعية "الحركات"** مخصّصة لتبديل أنواع الحركات الفردية:
  **فتح التطبيق**، و**إغلاق التطبيق / العودة**، و**القوائم وقائمة الإعدادات**.

</details>

<details>
<summary><b>📂 درج التطبيقات الذكي والبحث</b> — اضغط للتوسيع</summary>

<br/>

- **شريط تمرير أبجدي A–Z بأسلوب Niagara** للتنقّل الفائق السرعة بين كل التطبيقات.
- **مجلدات** مع **وضع تحرير** لإعادة ترتيب التطبيقات عبر السحب والإفلات.
- **قائمة المفضّلة** مباشرةً على الشاشة الرئيسية — مع التسميات أو بدونها اختياريًا.
- **إطار مفضّلة قابل للتهيئة** — اختر نمط حدود (بلا، أسود، أبيض، لون مميّز، خفيف).
- **بحث هجين** مع اقتراحات ذكية.

</details>

<details>
<summary><b>🗂️ إدارة التطبيقات</b> — اضغط للتوسيع</summary>

<br/>

- **إلغاء تثبيت التطبيقات** مباشرةً من المشغّل.
- **اختصارات التطبيقات** عبر الضغط المطوّل (روابط عميقة مباشرة إلى وظائف التطبيق).
- قائمة سياقية لكل تطبيق للإجراءات السريعة.

</details>

<details>
<summary><b>🤙 الإيماءات وإجراءات الجهاز</b> — اضغط للتوسيع</summary>

<br/>

- **إيماءات قابلة للتهيئة** عبر **قائمة فرعية "الإيماءات"** مخصّصة: عيّن إجراءً بحرية
  لـ **النقر المزدوج** (ولـ **الهز**).
- الإجراءات المتاحة: **درج التطبيقات**، و**البحث**، و**قفل الشاشة**، و**الإشعارات**،
  و**المصباح**، و**الكاميرا**، و**عدم الإزعاج**، و**فتح تطبيق محدّد**، و**الإعدادات**.
- **إيماءات التمرير:** اسحب للأعلى لدرج التطبيقات، وللأسفل للوحة الإشعارات.
- **تكامل الإشعارات** على الشاشة الرئيسية.

</details>

> ℹ️ **قسم المعلومات** — وصول شفّاف إلى إصدار التطبيق والتراخيص ومعلومات المطوّرين.

- JDK 17
- غلاف/خادم Gradle أيضًا على JDK 17

</div>

```

</div>

```

<div dir="rtl">

### ⚡ اختبار تطبيق الإصدار — سريع وعبر Wi-Fi

يعمل بناء الإصدار مع R8/minify ودون عبء `debuggable`، لذا فهو **أسرع بوضوح** من بناء التصحيح. وهو موقّع بمفتاح التصحيح ← قابل للتثبيت عبر Wi-Fi وكتحديث في المكان *بجانب* بناء التصحيح (دون الحاجة لإلغاء التثبيت).

</div>

```bash
# 1) صِل هاتفك عبر تصحيح Wi-Fi (أندرويد 11+)
#    الرموز من: خيارات المطوّر → تصحيح لاسلكي → إقران الجهاز برمز
adb pair    <ip-الهاتف>:<منفذ-الإقران> <رمز-الإقران>
adb connect <ip-الهاتف>:<منفذ-التصحيح>
adb devices                                   # ينبغي أن يظهر الجهاز

# 2) بناء وتثبيت الإصدار
./gradlew installRelease

# 3) تشغيل المشغّل (أو اضغط زر الشاشة الرئيسية فقط)
adb shell am start -n com.example.androidlauncher/.MainActivity
```

<div dir="rtl">

> 💡 لأن ProGuard/R8 مفعّل، تصفّح التطبيق بالكامل مرّة بعد بناء الإصدار. إذا كان هناك صنف مفقود أثناء التشغيل، أضِف قاعدة `-keep` مناسبة في `app/proguard-rules.pro`.

## 🩺 استكشاف الأخطاء

- ينجح `./gradlew clean build` حتى لو أبلغ `stripDebugDebugSymbols`/`stripReleaseDebugSymbols` بأن بعض ملفات `.so` لا يمكن تجريدها. في هذا السياق هي مجرّد تحذيرات.
- إذا توقّف `./gradlew clean run` سابقًا بالخطأ `Task 'run' not found`: مهمة الجذر `run` موجودة الآن كاسم بديل لـ `:app:installDebug`.
- استخدم دائمًا الغلاف (`./gradlew`) بدلًا من `gradle`، حتى يُستخدم الإصدار المثبّت في `gradle/wrapper/gradle-wrapper.properties`.
- تحذير `HAPTIC_FEEDBACK_ENABLED` في `ThemeManager.kt` هو تحذير إهمال ولا يمنع البناء.
- تحذيرات إهمال AGP العديدة أثناء `installRelease` غير ضارّة — طالما ظهر `BUILD SUCCESSFUL` فكل شيء على ما يُرام.
- لتحذيرات الإضافات/Gradle المفصّلة:

</div>

```bash
./gradlew --warning-mode all help
```
